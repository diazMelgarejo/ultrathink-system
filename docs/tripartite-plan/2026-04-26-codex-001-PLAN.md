# Hardware-Bound Model Routing Repair Plan

## Summary
Root cause: `orama-system/scripts/discover.py` trusts whatever `/v1/models` returns per LM Studio endpoint and writes those raw lists into `~/.openclaw/openclaw.json`, `discovery.json`, and `.env.lmstudio`. That lets model lists bleed across providers when LM Studio/LM Link exposes mixed models. The live config already shows this pollution: `lmstudio-mac` advertises Windows-only 27B/26B models, and `lmstudio-win` advertises Mac-only 9B MLX/Gemma E4B.

Canonical source already exists mostly in `Perpetua-Tools`: `hardware/SKILL.md`, `docs/MODEL_HARDWARE_MATRIX.md`, `config/models.yml`, and the older root-level `CORRECT AI-Agent-Model-Hardware-Matrix-v0.9.9.5.md`. The missing piece is a hard machine-enforced allow/deny policy and cross-repo visibility.

## Model Policy
- Windows-only forever: `gemma-4-26b-a4b-it`, `gemma-4-26B-A4B-it-Q4_K_M`, `Qwen3.5-27B-Claude-4.6-Opus-Reasoning-Distilled-v2`, `qwen3.5-27b-claude-4.6-opus-reasoning-distilled-v2`.
- Mac-only forever: `gemma-4-e4b-it`, `Qwen3.5-9B-MLX-4bit`, `qwen3.5-9b-mlx`, `qwen3.5-9b-mlx-4bit`.
- Shared/neutral models remain allowed only if explicitly listed as shared fallback, not inferred from discovery.
- OpenClaw provider lists must be filtered by provider before write: `lmstudio-mac` only receives Mac-allowed models, `lmstudio-win` only receives Windows-allowed models.

## Key Changes
- Add a canonical policy section to `Perpetua-Tools/hardware/SKILL.md` and `Perpetua-Tools/docs/MODEL_HARDWARE_MATRIX.md` explicitly naming the Windows-only and Mac-only model IDs.
- Add a small machine-readable policy in `Perpetua-Tools/config/model_hardware_policy.yml` or equivalent, then have docs reference it as the source for enforcement.
- Update `orama-system/scripts/discover.py` to filter discovered model lists through the same policy before patching `~/.openclaw/openclaw.json`, `discovery.json`, `last_discovery.json`, and `.env.lmstudio`.
- Add tests in `orama-system/scripts/tests/test_discover.py` proving mixed discovery input is split safely by hardware profile and never writes Windows-only models into `lmstudio-mac`.
- Cross-reference the policy in `orama-system/AGENT_RESUME.md`, `orama-system/docs/LESSONS.md`, `Perpetua-Tools/AGENT_RESUME.md`, and `Perpetua-Tools/docs/LESSONS.md`.
- Fix related Perpetua config drift while there: malformed `LM_STUDIO_WIN_ENDPOINTS` default and stale `win-rtx3080` LAN IP values should not point at the Mac endpoint.
