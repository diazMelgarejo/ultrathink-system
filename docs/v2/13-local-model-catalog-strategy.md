# 13 — Local Model Catalog Strategy (Codex CLI + Ollama)

> **Status**: Active | Added 2026-05-07
> **Scope**: `~/.codex/` global config + both repos that embed Codex agent configs
> (`.codex/` in `orama-system` and `Perpetua-Tools`).
> **Trigger**: `qwen3.5-local:latest` deleted from Mac Ollama; replaced by
> `qwen3.5:9b-nvfp4`. Codex CLI unknown-model fallback (272K default context)
> causing context-overflow and plan-mode failures.

---

## 1. The problem Codex solves with `model_catalog_json`

Codex CLI ships a built-in `models.json` catalog of known OpenAI cloud models.
When you run it against a local Ollama endpoint with a model it doesn't recognise,
it emits a warning and falls back to generic defaults — most critically:

- **Context window**: defaults to 272,000 tokens.
  Local models don't report their context size via the API, so Codex never
  compacts. Eventually the local server returns a context-overflow error.
- **Plan mode / structured output / tools**: may silently break because the
  flags (`supports_reasoning`, `supports_tools`) are unknown.

The fix: point `model_catalog_json` at a local JSON file that supplies the
missing metadata.

---

## 2. Canonical file layout

```
~/.codex/
├── config.toml           ← global; [model_providers.ollama-launch] now has
│                            model_catalog_json = "~/.codex/local_models.json"
└── local_models.json     ← single source of truth for all local model metadata
```

`local_models.json` is **not committed to any repo** — it lives only on the
developer's machine. It is regenerated by `scripts/gen_local_models_json.py`
(see §5).

---

## 3. Metadata values to set correctly

| Field | Source | Notes |
|-------|--------|-------|
| `context_window` | `curl localhost:11434/api/show` → `model_info.<family>.context_length` | Cap at a safe local value (≤ 65536 for 9B, ≤ 32768 for 7B) even if native is larger |
| `max_output_tokens` | Typically ¼ of `context_window` | |
| `supports_reasoning` | True only for thinking/CoT models | `qwen3.5:9b-nvfp4` = true; `qwen2.5-coder:7b` = false |
| `supports_tools` | True for all chat models that accept tool_calls | Embedding + reranker models = false |
| `supports_streaming` | True for all chat models | |

### Why cap context at a smaller value than native?

`qwen3.5:9b-nvfp4` reports `context_length = 262144` from Ollama but running
a 9B model at 256K context on a Mac will thrash memory or hang. Cap at 32768
(safe for most Mac RAM configs) and raise it only after confirming the machine
can handle it. The `_notes` field in `local_models.json` documents the native
value for reference.

## 5. `scripts/gen_local_models_json.py` — auto-generation

Create this script in `orama-system/scripts/` to regenerate
`~/.codex/local_models.json` whenever the Ollama model list changes:

```python
#!/usr/bin/env python3
"""
gen_local_models_json.py — regenerate ~/.codex/local_models.json from live Ollama.

Usage:
    python3 scripts/gen_local_models_json.py > ~/.codex/local_models.json

Requires: Ollama running at http://localhost:11434
"""
import json
import urllib.request
from datetime import date

CLOUD_SUFFIXES = (":cloud",)
SAFE_CONTEXT_CAP = {
    # model-family → safe local cap (tokens). Raise if machine can handle more.
    "qwen3_5":   32768,
    "qwen2":     32768,
    "nomic-bert": 2048,
}
DEFAULT_CAP = 32768

def get_tags():
    with urllib.request.urlopen("http://localhost:11434/api/tags") as r:
        return json.loads(r.read())["models"]

def get_model_info(name):
    data = json.dumps({"name": name}).encode()
    req = urllib.request.Request(
        "http://localhost:11434/api/show",
        data=data, method="POST",
        headers={"Content-Type": "application/json"},
    )
    with urllib.request.urlopen(req) as r:
        return json.loads(r.read())

def is_cloud(name):
    return any(name.endswith(s) for s in CLOUD_SUFFIXES)

def is_embed_or_rerank(family, name):
    return any(x in (family + name).lower() for x in ("embed", "rerank", "nomic-bert"))

def build_entry(name, info):
    details = info.get("details", {})
    model_info = info.get("model_info", {})
    family = details.get("family", "")

    # Resolve native context length
    native_ctx = None
    for k, v in model_info.items():
        if "context_length" in k:
            native_ctx = v
            break

    cap = SAFE_CONTEXT_CAP.get(family, DEFAULT_CAP)
    context_window = min(native_ctx, cap) if native_ctx else cap
    embed = is_embed_or_rerank(family, name)
    cloud = is_cloud(name)

    return {
        "id": name,
        "name": f"{name} ({'cloud proxy' if cloud else details.get('quantization_level','local')})",
        "context_window": 131072 if cloud else context_window,
        "max_output_tokens": 32768 if cloud else context_window // 4,
        "supports_reasoning": not embed,
        "supports_tools": not embed,
        "supports_streaming": not embed,
        "_notes": f"native_context={native_ctx}; family={family}; generated {date.today()}",
    }

models = []
for tag in get_tags():
    name = tag["name"]
    try:
        info = get_model_info(name)
        models.append(build_entry(name, info))
    except Exception as e:
        print(f"# WARN: could not get info for {name}: {e}", flush=True)

print(json.dumps({"_comment": f"Auto-generated {date.today()} — do not edit manually", "models": models}, indent=2))
```

## 6. Adoption in v2 (`perpetua-core`)

The `model_hardware_policy.yml` + `local_models.json` pair maps to
**Pattern B** from `11-idempotency-and-guard-patterns.md`:
one config file consumed by multiple validators / clients.

In v2:

```
perpetua-core/config/
├── model_hardware_policy.yml      ← dispatch policy (ALLOW/PREFER/NEVER per tier)
└── ollama_catalog.json            ← mirrors ~/.codex/local_models.json schema;
                                     used by HardwarePolicyResolver + Codex agents
```

`ollama_catalog.json` is gitignored by default (machine-specific). An
`ollama_catalog.example.json` ships with the repo as a template. CI uses the
example file. Developers generate their own with `gen_local_models_json.py`.

RETIRED_IDS = {
    "qwen3.5-local:latest",
